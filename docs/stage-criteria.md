# Stage Criteria: G42 SaaS Framework

Detailed criteria and checklists for each stage of the SaaS Framework governance process.

---

## Stage 1: IDEATE

### Purpose
Generate a long list of ideas with clear outline of expected business value.

### Idea Types
| Type | Description |
|------|-------------|
| **Greenfield** | Completely new product idea |
| **Brownfield** | SaaSify an existing project/solution |
| **White-label** | Partner white-labeling opportunity |

### Entry Criteria
- Idea submitted via approved channel (MS Forms or GitHub template)
- BU sponsor identified

### Checklist
- [ ] **Clear problem statement defined**
  - What problem does this solve?
  - Who has this problem?
  - How significant is the problem?

- [ ] **Business value articulated**
  - What is the expected revenue potential?
  - What strategic value does this provide?
  - What competitive advantage does this create?

- [ ] **Initial market sizing done**
  - Total Addressable Market (TAM)
  - Serviceable Addressable Market (SAM)
  - Serviceable Obtainable Market (SOM)

- [ ] **BU sponsor identified**
  - Named executive sponsor
  - Committed resources for evaluation

- [ ] **Group template filled**
  - All required fields completed
  - Supporting documentation attached

### Exit Criteria
- All checklist items complete
- Admin approval for transition to Validate

## Stage 2: VALIDATE

### Purpose
Prioritize ideas based on G42 market potential and effort assessments.

### Stakeholder Responsibilities
| Stakeholder | Role |
|-------------|------|
| **BUs** | Provide information as required |
| **Group** | Evaluate against validation engine |
| **MSFT** | Support information provision |

### Validation Engine Criteria

#### Business Value Assessment
| Score | Revenue Potential | Strategic Alignment |
|-------|-------------------|---------------------|
| High | >$10M ARR | Core to G42 strategy |
| Medium | $1M-$10M ARR | Aligned with strategy |
| Low | <$1M ARR | Opportunistic |

#### Effort Assessment
| Factor | Considerations |
|--------|----------------|
| **Sales** | GTM complexity, channel readiness |
| **Technical** | Development effort, infrastructure needs |
| **Funding** | Investment required, ROI timeline |

### Checklist
- [ ] **Market potential assessed**
  - Market size validated
  - Growth trajectory analyzed
  - Competitive landscape reviewed

- [ ] **Effort estimation completed**
  - Development timeline estimated
  - Resource requirements defined
  - Budget allocated

- [ ] **Technical feasibility reviewed**
  - Architecture approach defined
  - Technology stack selected
  - Integration requirements identified

- [ ] **Business case documented**
  - Financial projections completed
  - Risk assessment done
  - Success metrics defined

- [ ] **MSFT validation obtained**
  - Global vertical knowledge applied
  - SME review completed

- [ ] **Go/No-Go decision made**
  - Steering committee approval
  - Priority ranking assigned

### Exit Criteria
- Positive Go decision
- Prioritization complete
- Resources allocated for Build

---

## Stage 3: BUILD

### Purpose
Develop functional SaaS products ready to market.

### Stakeholder Responsibilities
| Stakeholder | Role |
|-------------|------|
| **BUs** | Lead product execution |
| **Group** | Allocate ISE Engineers |
| **MSFT** | Co-build, ensure PLG execution |

### Checklist
- [ ] **Development team assigned**
  - Core team identified
  - ISE Engineers allocated
  - MSFT co-build resources engaged

- [ ] **Technical architecture approved**
  - Architecture design reviewed
  - Security assessment completed
  - Scalability plan approved

- [ ] **MVP scope defined**
  - Feature prioritization done
  - User stories documented
  - Acceptance criteria defined

- [ ] **Sprint plan created**
  - Release milestones set
  - Sprint cadence established
  - Demo schedule confirmed

- [ ] **MVP developed**
  - Core features implemented
  - Integration completed
  - Performance optimized

- [ ] **Testing completed**
  - Unit testing done
  - Integration testing done
  - User acceptance testing passed

- [ ] **Documentation ready**
  - User documentation complete
  - API documentation complete
  - Operations runbook created

### Exit Criteria
- Product is functional
- Quality standards met
- Ready for market release

---

## Stage 4: SELL

### Purpose
Package product offering with services & add-ons.

### Stakeholder Responsibilities
| Stakeholder | Role |
|-------------|------|
| **BUs** | Lead product sales with Group coordination |
| **Group** | Lead strategic int'l markets and I3 accounts |
| **MSFT** | Activate in Azure Marketplace |

### Checklist
- [ ] **Pricing model defined**
  - Pricing tiers established
  - Licensing model confirmed
  - Discount policies set

- [ ] **Sales collateral created**
  - Product datasheets
  - Demo scripts
  - Case studies (if available)

- [ ] **Azure Marketplace listing** (if applicable)
  - Listing created and approved
  - Transact capability enabled
  - Co-sell ready status achieved

- [ ] **Partner channel activated**
  - Partner agreements in place
  - Partner enablement completed
  - Joint marketing planned

- [ ] **First customers acquired**
  - Pilot customers onboarded
  - Reference customers identified
  - Revenue recognized

### Exit Criteria
- Product actively being sold
- Revenue being generated
- Customer base established

---

## Stage 5: REVIEW

### Purpose
Assess product performance and iterate to improve.

### Stakeholder Responsibilities
| Stakeholder | Role |
|-------------|------|
| **BUs** | Track Success KPIs, support feedback loops |
| **Group** | Define/standardize KPIs, support feedback |
| **MSFT** | Utilize Unified Support for similar services |

### Key Performance Indicators (KPIs)

| Category | Metrics |
|----------|---------|
| **Revenue** | ARR, MRR, growth rate |
| **Customers** | Count, retention, NPS |
| **Product** | Usage, adoption, feature utilization |
| **Operations** | Uptime, support tickets, resolution time |

### Checklist
- [ ] **KPIs defined and tracked**
  - Metrics dashboard created
  - Baseline measurements taken
  - Targets established

- [ ] **Customer feedback collected**
  - NPS surveys conducted
  - Feature requests compiled
  - Support trends analyzed

- [ ] **Performance review completed**
  - Quarterly review held
  - Results vs targets analyzed
  - Lessons learned documented

- [ ] **Improvement roadmap created**
  - Enhancement priorities set
  - Technical debt addressed
  - Innovation opportunities identified

- [ ] **Next iteration planned**
  - Version roadmap updated
  - Resource allocation confirmed
  - Timeline established

### Exit Criteria
- Continuous cycle established
- Regular review cadence maintained
- Improvement actions tracked

## Stage Transition Summary

| From | To | Required |
|------|-----|----------|
| Ideate | Validate | All Ideate checklist complete, Admin approval |
| Validate | Build | Go decision, Priority assigned, Resources allocated |
| Build | Sell | MVP complete, Quality approved, Documentation ready |
| Sell | Review | Revenue generated, Customers onboarded |
| Review | (Continuous) | Ongoing performance monitoring |
