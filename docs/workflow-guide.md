# Workflow Guide: G42 SaaS Framework Process

This guide explains how to use the G42 SaaS Factory Board to manage ideas through the governance process.

## Overview

The SaaS Framework has 5 stages, each with specific stakeholder responsibilities:

```
IDEATE → VALIDATE → BUILD → SELL → REVIEW
```

## Roles & Responsibilities

### Business Units (BUs)
- **Lead**: Ideate, Build, Sell stages
- **Support**: Validate, Review stages
- Generate and champion ideas
- Execute product development
- Drive sales within their markets

### Group
- **Lead**: Validate, Review stages
- **Support**: Ideate, Build, Sell stages
- Provide templates and evaluation criteria
- Allocate centralized build resources
- Define success KPIs

### Microsoft (MSFT)
- **Support**: All stages
- Facilitate CEO roundtables for ideation
- Validate ideas via Global vertical knowledge/SMEs
- Co-build with ISE Engineers
- Activate products in Azure Marketplace

---

## Stage Details

### 1. IDEATE Stage

**Outcome:** Long list of ideas with clear outline of expected business value

**How to submit an idea:**
1. Fill out the Microsoft Forms nomination form, OR
2. Create a GitHub issue using the "Idea Nomination" template

**Checklist to complete:**
- [ ] Clear problem statement defined
- [ ] Business value articulated
- [ ] Initial market sizing done
- [ ] BU sponsor identified
- [ ] Group template filled

**Exit criteria:** All checklist items complete, admin approves transition

---

### 2. VALIDATE Stage

**Outcome:** Prioritized ideas based on G42 market potential and effort assessments

**Activities:**
- Group evaluates against validation engine
- Business value ($) assessment
- Effort estimation (sales, technical, funding)
- MSFT provides validation support

**Checklist to complete:**
- [ ] Market potential assessed
- [ ] Effort estimation completed
- [ ] Technical feasibility reviewed
- [ ] Business case documented
- [ ] MSFT validation obtained
- [ ] Go/No-Go decision made

**Exit criteria:** Positive Go decision, prioritization complete

---

### 3. BUILD Stage

**Outcome:** Developed functional SaaS products ready to market

**Activities:**
- BU leads product execution
- Group allocates ISE Engineers
- MSFT provides co-build support
- Product-Led Growth execution

**Checklist to complete:**
- [ ] Development team assigned
- [ ] Technical architecture approved
- [ ] MVP scope defined
- [ ] Sprint plan created
- [ ] MVP developed
- [ ] Testing completed
- [ ] Documentation ready

**Exit criteria:** Product is functional and ready for market

---

### 4. SELL Stage

**Outcome:** Packaged product offering with services & add-ons

**Activities:**
- BU leads sales in coordination with Group
- Group leads strategic international markets
- MSFT activates in Azure Marketplace

**Checklist to complete:**
- [ ] Pricing model defined
- [ ] Sales collateral created
- [ ] Azure Marketplace listing (if applicable)
- [ ] Partner channel activated
- [ ] First customers acquired

**Exit criteria:** Product actively being sold

---

### 5. REVIEW Stage

**Outcome:** Assess product performance and iterate to improve

**Activities:**
- BU tracks Success KPIs
- Group defines/standardizes KPIs across BUs
- MSFT supports through Unified Support contract

**Checklist to complete:**
- [ ] KPIs defined and tracked
- [ ] Customer feedback collected
- [ ] Performance review completed
- [ ] Improvement roadmap created
- [ ] Next iteration planned

**Exit criteria:** Continuous cycle of review and improvement

---

## Using the GitHub Project Board

### Viewing Ideas
1. Go to the repository's **Projects** tab
2. Open the **SaaS Factory Board**
3. Ideas are organized by stage columns

### Filtering
- Click **Filter** to filter by:
  - Business Unit
  - Stage
  - Idea Type
  - Priority

### Moving Ideas Between Stages (Admins)
1. Ensure all checklist items are complete
2. Look for the `status:ready-for-review` label
3. Drag the issue to the next stage column, OR
4. Update the stage label manually

---

## Working on an Idea

### Adding Comments
- Use issue comments for discussion
- Tag stakeholders using @mentions
- Reference related issues with #issue-number

### Uploading Documents
- Drag and drop files into issue comments
- Or use the attach button in the comment box
- Supported: PDFs, images, Office documents

### Updating Checklists
- Click the checkbox in the issue body to toggle
- When all boxes are checked, the system adds `status:ready-for-review`

---

## Notifications

GitHub will notify you when:
- You're mentioned in a comment
- An issue you're watching is updated
- You're assigned to an issue

Configure notifications in your GitHub settings.
